# SauceDemo Test Plan

## 1. Objective

The purpose of this test plan is to verify the core functionality, usability, and stability of the SauceDemo web application.

---

## 2. Scope

The following areas will be tested:

- Login functionality
- Logout functionality
- Inventory page
- Product sorting
- Add to cart functionality
- Remove from cart functionality
- Shopping cart behavior
- Session management
- Checkout process
- Form validation
- Navigation between pages
- Error messages
- UI consistency

---

## 3. Test Types

The following testing types will be performed:

- Functional Testing
- Smoke Testing
- Regression Testing
- Exploratory Testing
- Negative Testing
- UI Testing
- Validation Testing
---

## 4. Features to be Tested

| Feature | Description |
|---|---|
| Login | Authentication using valid and invalid credentials |
| Product Inventory | Display and sorting of products |
| Shopping Cart | Adding and removing products |
| Checkout | Completing purchase flow |
| Form Validation | Validation of required fields |
| Navigation | Navigation between application pages |
| Error Handling | Proper display of validation and error messages |

---

# 5. Test Environment

| Component | Details |

| Application | SauceDemo |

---

## 6. Test Strategy

Testing will be performed manually using predefined test cases and exploratory testing techniques.

The strategy includes:

- positive and negative test scenarios
- validation testing
- business-critical user flows
- edge case analysis

Priority will be given to critical functionalities such as authentication and checkout process.

---

## 7. Entry Criteria

Testing activities may begin when:

- application is accessible
- test environment is available
- test data is prepared
- testing scope is defined

---

## 8. Exit Criteria

Testing activities may be completed when:

- planned test cases are executed
- critical defects are documented
- major functionalities are verified
- test results are reviewed
